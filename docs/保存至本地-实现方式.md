# 「保存至本地」后续功能实现方式

当前点击「保存至本地」会调用 `POST /api/records/save`，将病历写入 **Supabase**（服务端数据库）。以下是几种可选的实现方式，可按需求扩展。

---

## 1. 当前方式：后端 + Supabase（已实现）

- **流程**：前端 → `POST /api/records/save` → `server/lib/records-db.ts` → Supabase 表 `medical_records`。
- **特点**：多端同步、可做病历列表/检索、需要网络；未配置 Supabase 时保存会落空。
- **适用**：正式环境、多设备、需要后台管理。

---

## 2. 纯前端本地存储（localStorage / IndexedDB）

- **实现**：不调后端，在前端用 `localStorage` 或 `IndexedDB` 存病历 JSON；可建 key 如 `medical_drafts` 存数组。
- **特点**：不依赖网络、数据仅当前浏览器/设备；清缓存会丢失。
- **适用**：草稿箱、离线优先、或后端未就绪时的兜底。
- **扩展**：可做「草稿箱」页，从本地列表读取、编辑、再提交到后端。

---

## 3. 导出为文件（下载 JSON / PDF）

- **实现**：把当前病历（含主诉、评估、计划等）组装成 JSON 或生成 PDF，触发浏览器下载（如 `Blob` + `a.download`）。
- **特点**：用户主动「另存为」到设备，便于备份、归档或导入其他系统。
- **适用**：需要留档、打印、或与院外系统对接。

---

## 4. 离线优先（IndexedDB + 有网时同步）

- **实现**：先写入本地 IndexedDB，同时入队「待同步」；有网时轮询或监听 online 事件，将待同步记录 `POST /api/records/save`，成功后从队列移除。
- **特点**：离线可存、联网后自动同步，兼顾无网与多端。
- **适用**：弱网/现场无网场景。

---

## 5. 微信小程序侧（本地 + 可选云）

- **本地**：`wx.setStorageSync` / `wx.getStorageSync` 存病历对象或列表；仅当前设备。
- **云开发**：使用微信云开发数据库，实现多端同步与后台管理。
- **适用**：小程序独立使用或与现有后端并存。

---

## 建议组合

- **保留**：现有「保存至后端 Supabase」逻辑。
- **增强**：  
  - 增加 **localStorage 兜底**：请求失败时自动写入本地并提示「已保存至本地草稿箱（未同步到服务器）」；和/或  
  - 增加 **导出为文件**：提供「导出 JSON」按钮，一键下载当前病历。

这样「保存至本地」可以有多种含义：存到服务器、存到浏览器本地、或导出到文件，后续再按需扩展草稿箱列表与离线同步。
